# Specification: Google OAuth Authentication

**Track ID:** google-auth_20260407
**Type:** Feature
**Created:** 2026-04-07

## Summary

Implement Google OAuth authentication via Supabase Auth with role-based access control (Guest/Player/Admin) and corresponding UI in the header.

## Context

The app has three user roles defined in product.md: Guest (view only), Player (self-service game participation), Admin (full CRUD). The `profiles` table with auto-creation trigger and RLS policies are already in place (migrations 002, 005). Layout (AppHeader, default.vue) exists from project-scaffold track. `@nuxtjs/supabase` module is installed and configured with `redirect: false`.

## User Story

As a player, I want to sign in with my Google account so that I can join games, choose my role, and track my personal statistics.

## Acceptance Criteria

- [ ] Guest sees "Sign In" button in AppHeader
- [ ] Clicking "Sign In" initiates Google OAuth flow via Supabase
- [ ] After successful login, user sees their avatar and nickname in AppHeader
- [ ] Logged-in user has a dropdown menu with profile link and "Sign Out"
- [ ] Admin users see an admin badge on their avatar
- [ ] `auth` middleware redirects unauthenticated users to home page
- [ ] `admin` middleware redirects non-admin users to home page
- [ ] Profile is auto-created on first login (existing DB trigger)
- [ ] `useAuth` composable exposes: user, profile, isAuthenticated, isAdmin, signIn, signOut
