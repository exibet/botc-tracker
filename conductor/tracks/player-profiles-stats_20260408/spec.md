## Summary

Player profiles page with detailed game statistics and game history. Includes a players list grid and individual profile pages with stats cards, role distribution chart, and game history table.

## Context

The BotC Tracker already has games CRUD, player management, and game_players records. This track adds the player-facing views: a browseable players grid and per-player profile pages with aggregated statistics. Target community is ~8-15 regular players, up to 100 max.

## User Story

As a player or guest, I want to view player profiles with detailed game statistics (win rate, MVP count, survival rate, favourite roles) and game history so that I can track progress and compare performance within the community.

## Acceptance Criteria

- [ ] `/players` page shows a grid of all players with avatar, nickname, games count, and win rate
- [ ] `/players/[id]` page displays full profile: header, 6 stat cards (games, win rate, MVPs, survival, good wins, evil wins)
- [ ] Role distribution doughnut chart shows roles played by type
- [ ] Game history table lists all games for that player with date, script, role, alignment, status, result
- [ ] Stats are calculated via Supabase queries aggregating `game_players` + `games` data
- [ ] Responsive layout: cards on mobile, full tables/charts on desktop
